# `Remote`

The `Remote` namespace is the **wallet-side** counterpart to the [`Provider`](/docs/api/provider). It:

- runs inside the iframe or popup served by the wallet's auth app
- receives RPC requests forwarded from the host SDK over [`Messenger`](https://github.com/tempoxyz/accounts/blob/main/src/core/Messenger.ts)
- exposes a queue of pending requests for the UI to render approval surfaces against

A typical wallet app calls [`Remote.create`](/docs/api/remote.create) once at startup, subscribes to the request queue with `onUserRequest`, navigates to a per-method route, and either [`respond`](#respond) or [`reject`](#reject)s once the user has approved or cancelled.
