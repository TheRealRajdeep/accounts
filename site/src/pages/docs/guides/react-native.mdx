# React Native

Connect account flows from an Expo or React Native app while accounting for mobile storage, browser, and passkey differences.

## Demo

Target workspace: `examples/react-native/`.

Seed implementation: `playgrounds/react-native/`.

:::warning[Stub]
This guide needs a mobile pass before it moves from playground to example. Keep platform caveats explicit because React Native differs from browser WebAuthn and IndexedDB behavior.
:::

## Walkthrough

::::steps

### Install the SDK

```bash
pnpm add accounts expo-secure-store expo-web-browser
```

### Configure the Provider

Use the React Native entrypoint and storage appropriate for the app.

### Connect the user's account

Open the wallet or auth flow from the mobile shell.

### Submit the request

Submit a simple account request and handle mobile browser return state.

### Verify the result

Test iOS and Android behavior before publishing the guide.

::::

## Notes / Gotchas

- React Native can expose `window` without browser APIs such as `CustomEvent` or IndexedDB.
- Expo playgrounds in this workspace need local app entrypoints and Metro-safe package resolution.
