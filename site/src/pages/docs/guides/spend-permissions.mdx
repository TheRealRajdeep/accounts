# Spend Permissions

Authorize scoped account access so an app or service can perform repeat activity without prompting the user for every request.

## Demo

Target workspace: `examples/spend-permissions/`.

Seed implementations: `examples/access-key/` and `examples/access-key-and-webauthn/`.

## Walkthrough

::::steps

### Install the SDK

```bash
pnpm add accounts wagmi viem
```

### Configure the Provider

Use the adapter that owns user approval for the access request.

### Connect the user's account

Connect the account before requesting scoped access.

### Submit the request

Call the access-key authorization flow with the intended expiry and permissions.

### Verify the result

Store the returned access key material only in the environment that is allowed to use it, then test revocation.

::::

## Best Practices

- Keep permissions as narrow as the workflow allows.
- Prefer short expiries for automation and longer-lived access only after explicit user approval.
- Show users where they can revoke access.
